# Политика подагентов Codex

Этот файл описывает, как в этом проекте применять подагентов Codex. Он дополняет
`AGENTS.md`: там указано, когда читать этот файл, здесь — как выполнять
маршрутизацию.

## Источник настройки

Общие параметры fan-out и ограничений запуска закреплены в пользовательской
конфигурации Codex `~/.codex/config.toml`. Проект использует этот глобальный
слой как основной источник настройки подагентов.

Перед делегированием сначала проверь пользовательский слой `~/.codex/config.toml`
и глобальные определения агентов `~/.codex/agents/*.toml`.

## Модели и лимиты

- `cheap_explorer` и `cheap_patch_worker` работают на
  `gpt-5.3-codex-spark`. Эта модель использует отдельный лимит и не расходует
  общий лимит `gpt-5.4`/`gpt-5.4-mini`/`gpt-5.5`.
- `bounded_policy_reviewer` работает на `gpt-5.4-mini`. Модель расходует общий
  лимит вместе с `gpt-5.4` и `gpt-5.5`, но должна тратить его медленнее, чем
  более сильные модели.
- `strong_reviewer` работает на `gpt-5.5` и расходует общий лимит подписки
  «ChatGPT Pro 5x».

Для правильной классификации лимитов в локальном отчёте задавай переменные:

```bash
CODEX_COMMON_LIMIT_MODELS=gpt-5.4,gpt-5.4-mini,gpt-5.5
CODEX_SEPARATE_LIMIT_MODELS=gpt-5.3-codex-spark
```

## Выбор маршрута

Не запускай подагентов для микрозадач, правок на 1–2 локальных шага и задач, где
контракт и проверка результата дороже прямого выполнения родителем.

Предпочитай `cheap_explorer` для первого ограниченного шага без записи:

- поиск по нескольким файлам;
- сбор фактов;
- проверка охвата;
- сравнение малого набора артефактов;
- подготовка кандидатов на чтение.

Считай запуск подагента обязательным только если пользователь или проектное
правило прямо требуют подагента, либо задача проверяет саму схему подагентов.
Если запуск обязателен, недоступность роли, модели, отдельного лимита или
проверяемого технического следа — блокер до разрешения человека.

Не запускай больше одного подагента на один шаг родителя и не допускай
рекурсивный fan-out через дочерние потоки.

## Гарантированный выбор модели

Если нужно гарантированно выбрать модель подагента, запускай отдельный процесс
Codex:

```bash
tools/codex-model-subagent <model> <name> <prompt>
```

Для максимальной экономии общего лимита используй:

```bash
tools/codex-model-subagent gpt-5.3-codex-spark <name> <prompt>
```

Запускатель возвращает JSONL запуска, дочерний session JSONL, модель, лимиты,
время и финальный результат.

## Проверка технического следа

Перед тем как считать делегирование экономичным, проверь технический след:

- для встроенного fan-out — `spawn_agent`/`wait`, дочерний session JSONL, модель
  дочернего потока и `rate_limits.limit_id`;
- для `tools/codex-model-subagent` — JSONL запуска, дочерний session JSONL,
  модель, лимиты, время и финальный результат.

Текст «я использовал подагента» не является доказательством.

Если fan-out породил дочерний поток с `agent_path = null` или моделью родителя,
это не считается запуском `cheap_explorer` и не считается экономией общего
лимита.

## Роли

`cheap_explorer` используй первым для узкого поиска, извлечения фактов,
кандидатов на чтение и коротких сводок со ссылками на файлы.

`cheap_patch_worker` используй только для малых обратимых правок с явно заданной
областью записи. Ему запрещено менять `AGENTS.md`, навыки, правила, шаблоны,
требования, архитектурные решения и другие источники истины.

`bounded_policy_reviewer` используй для ограниченной независимой проверки
небольшого набора уже выбранных файлов. Если меняются правила, навыки, шаблоны,
README, архитектурные решения или другие источники истины, и набор файлов уже
ограничен, по умолчанию рассматривай `bounded_policy_reviewer` как первый
независимый обзор до `strong_reviewer`.

`strong_reviewer` используй редко, только перед ответственными изменениями
правил, навыков, шаблонов и других источников истины, когда проверка на
`gpt-5.4-mini` недостаточна.

## Ограничения и приёмка

По умолчанию подагентам запрещено редактировать файлы. Исключение —
`cheap_patch_worker`, если родитель явно указал файлы или узкую область записи и
затем сам проверяет diff, запускает доступную проверку и принимает или отклоняет
результат.

Всем подагентам запрещено принимать финальные проектные решения, утверждать
изменения источников истины и расширять область задачи без явной эскалации.

Если результат подагента не даёт ссылок на файлы, содержит конфликтующие
свидетельства, требует больше одной повторной попытки или не даёт дешёвой
проверки, родитель обязан остановить делегирование и продолжить сам либо
эскалировать на более сильную модель.

Родительский агент обязан сам принимать результат подагента, проверять ссылки на
файлы, отличать факт от вывода и проходить `skills/ai-work-control` перед
изменением правил, навыков, шаблонов и других источников истины.

## Отчёт о расходе

После запуска подагентов через отдельный процесс запускатель автоматически
печатает одну строку расхода: модели, общий/отдельный лимит и время. Если строка
мешает автоматизации, отключай её через `CODEX_SUBAGENT_USAGE_LINE=0`.

Для общей сводки используй:

```bash
tools/codex-usage-report
```

Для подробной таблицы:

```bash
tools/codex-usage-report --details <jsonl-or-dir>
```

После задачи, где фактически запускались подагенты, добавляй в ответ
однострочную сводку моделей и лимитов.
