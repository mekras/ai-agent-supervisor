# ai-application-check

Навык применяется до выбора строгого маршрута, когда ещё неясно, подходит ли
задача для выполнения с помощью ИИ и нужен ли более сильный контроль.

## Обязательный порядок

1. Опиши задачу и ожидаемый результат.
2. Проверь, есть ли источники и контекст для корректного выполнения.
3. Определи, кого/что подключить: роли, навыки, процедуры.
4. Отметь границы автономности и проверки человеком.
5. Перечисли риски и короткий план их снижения.

## Результат проверки

Минимально вернуть:

- краткий ответ: `применять`, `пауза`, `переопределить задачу` или `отдать человеку`;
- список ключевых рисков;
- что осталось неопределённым;
- куда передавать на проверку после выполнения.

## Когда использовать

- Перед запуском анализа, когда задача неочевидна, спорна или контекст неполный.
- Перед проектным решением, если ещё неясно, нужен ли `ai-work-control`,
  `ai-data-collection` или передача человеку.
- Перед автоматизацией повторяющегося решения проекта.

## Когда не применять

- если требуется только правка существующего текста/кода без проектного эффекта;
- если задача явно меняет правила, роли, навыки, шаблоны, документацию или
  другой источник истины; тогда основной маршрут — `ai-work-control`;
- если задача уже строго попадает в `ai-data-collection` или `ai-work-control` по маршруту;
- если нужен только локальный `agents`/`tool` справочник без влияния на источник истины.

## Границы

- Не выполняет полноценную пост‑проверку качества результата; если результат важный,
  дальше подключается `ai-work-result-evaluation`.
- Не формирует структурный план работы проекта вместо самой проверки применимости.

Если задача уже относится к валидации результата, не запускай `ai-application-check` как основной этап.
