# Настройка подагентов и моделей

`ai-subagent-setup` помогает настроить экономное использование подагентов:
какие задачи отдавать быстрой модели, где нужна проверка, а где лучше оставить
работу основной модели.

Навык полезен, если вы хотите:

- меньше тратить основной лимит модели;
- не настраивать одних и тех же подагентов в каждом проекте заново;
- сохранить единые правила проверки результата;
- оставить возможность особых правил в отдельных проектах.

## Как попросить

Для обычной настройки:

```text
Используй $ai-subagent-setup.
Настрой экономное использование подагентов.
```

Если одна схема нужна в нескольких проектах на одной машине:

```text
Используй $ai-subagent-setup.
Настрой общую схему подагентов для моих проектов.
```

Если вы уже знаете доступные модели, можно перечислить их сразу:

```text
Используй $ai-subagent-setup.
Настрой общую схему подагентов для моих проектов.
У меня подписка «ChatGPT Pro 5x». Модели gpt-5.4, gpt-5.4-mini и gpt-5.5
используют общие лимиты, но с разной скоростью. У gpt-5.3-codex-spark лимиты
отдельные, её использование не влияет на основные лимиты. 
```

Если модели не указаны, агент сначала попробует найти их сам. Если не
получится, попросит список моделей.

## Глобальная настройка

Для повторяющейся схемы лучше использовать глобальную настройку Codex, а в
проектах хранить только отличия.

Так меньше копирования, меньше расхода токенов на повторную настройку и меньше
риск, что разные проекты постепенно разъедутся по правилам.

Если глобальной настройки ещё нет, а в проекте уже есть рабочая `.codex/`
конфигурация, при настройке появится предложение перенести общую часть в
глобальную настройку. В проекте после этого остаются только локальные
переопределения:
особые модели, запреты, команды проверки или правила приёмки.

Название и версию схемы агент подберёт сам и покажет в результате настройки.

## Как обновить

Если подагенты уже настроены, попросите обновить схему через текущую версию
навыка:

```text
Используй $ai-subagent-setup.
Обнови существующую настройку подагентов.
```

Агент сам проверит текущую схему, найдёт устаревшие проектные правила и
покажет, что можно перенести в глобальную настройку, а что лучше оставить в
конкретном проекте.

## Настройка одного проекта

Проектная настройка нужна, когда правила действительно отличаются от остальных
проектов: например, подагентам нельзя писать файлы, нужны другие модели или
есть особые источники истины.

Для Codex рабочая настройка обычно затрагивает глобальную конфигурацию Codex или
проектные файлы `.codex/config.toml`, `.codex/agents/*.toml` и короткие правила
в `AGENTS.md`.

## Как проверить

Для проверки текущей схемы:

```text
Используй $ai-subagent-setup.
Проверь текущую схему подагентов.
```

Настройка считается неполной, если есть только текстовые правила, но нет рабочей
конфигурации там, где среда её поддерживает, или если запуск подагента не
подтверждён фактической моделью и отчётом расхода.

Технические детали, шаблоны и порядок проверки Codex описаны в
`references/routing-patterns.md`.
