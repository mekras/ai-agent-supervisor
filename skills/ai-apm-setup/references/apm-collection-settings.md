# Настройки коллекции APM

Эту справку читай перед созданием или изменением `apm.yml` для коллекции
навыков.

## Минимальный манифест

Для коллекции навыков обычно нужны:

```yaml
name: project-name
version: 0.1.0
description: "Коллекция навыков"
target:
  - codex
type: "skill"
dependencies:
  apm: []
  mcp: []
scripts:
  tests: "..."
```

Если проект уже указывает `target`, сохрани его. Если целевая среда неясна, не
добавляй все возможные значения автоматически: возьми значение из проекта или
уточни у человека.

## Команда tests

`scripts.tests` должна запускать существующие проверки коллекции. Для
репозитория с валидаторами навыков используй:

```bash
sh -c 'set -e; target="${APM_EVAL_PATH:-skills}"; python3 tools/validate-trigger-evals.py "$target" --require-all; python3 tools/validate-skill-result-evals.py "$target"; python3 tools/run-skill-evals.py "$target"'
```

Если таких инструментов нет, найди существующую команду тестов проекта. Если
тестов нет, сначала создай минимальную проверку коллекции или остановись и
сообщи, что `apm run tests` нельзя честно настроить без команды тестов.

## Выборочный запуск

Команда `scripts.tests` для коллекции навыков должна оставлять полный запуск
значением по умолчанию, но поддерживать узкий цикл разработки:

- `apm run tests` — полный набор коллекции;
- `APM_EVAL_PATH=skills/<name> apm run tests` — проверки одного навыка;
- `APM_EVAL_PATH=skills/<name> APM_EVAL_CASE_ID=<id> apm run tests` — один
  модельный сценарий.

Выборочный запуск не заменяет полный запуск перед публикацией, изменением общей
инфраструктуры тестов или правкой нескольких навыков. Он нужен, чтобы быстро
проверять локальную правку и не тратить модельные вызовы на незатронутые
каталоги навыков.

## Проверка после правки

Минимально проверь:

- `apm.yml` существует в корне проекта;
- `type` соответствует публикуемой коллекции;
- `scripts.tests` есть и ссылается на существующие файлы или команды;
- `apm run tests` запускается либо объяснено, почему запуск сейчас невозможен;
- для выпуска выполнены `apm view`, `apm audit` и `apm pack`, если они доступны
  и входят в задачу.

`apm compile` не обязателен для пакета, который публикует только `skills/**`.
Он нужен, когда пакет содержит `instructions` или проект явно проверяет
компиляцию таких примитивов APM.
