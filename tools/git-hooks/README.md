# Git-хуки проекта

Каталог содержит отслеживаемые в Git хуки. Логика проверок вынесена в
скрипты `tools/`, поэтому хуки остаются тонкими обёртками.

## commit-msg

Проверяет первую строку сообщения коммита по детерминированным правилам
навыка `ru-dev`: язык, прописная буква в начале, форма результата вместо
инфинитива, отсутствие англоязычных шаблонов выпуска версии. Это внешний
контроль, который срабатывает одинаково для Claude Code, Codex и ручного
коммита и не зависит от того, применил ли агент навык.

Логика проверки: `tools/lint-commit-msg.py`.

## Активация

`core.hooksPath` — локальная настройка Git, она не хранится в репозитории.
В каждом клоне включи хуки один раз:

```sh
git config core.hooksPath tools/git-hooks
```

После этого Git берёт хуки из этого каталога вместо `.git/hooks`.

## Проверка вручную

```sh
printf 'Выпущена версия 0.5.0\n' | python3 tools/lint-commit-msg.py -
```

Код возврата `0` — нарушений нет, `1` — есть нарушения.
