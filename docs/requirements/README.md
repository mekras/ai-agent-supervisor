# Требования проекта

В этом каталоге хранятся принятые и предлагаемые требования к публикуемому
продукту проекта — коллекции `.apm/skills/**`.

## Правила ведения

- Каждое требование записывается отдельно и получает устойчивый идентификатор
  `REQ-NNNN`.
- Имя файла совпадает с идентификатором: `REQ-NNNN.md`.
- Запись содержит источник, уровень, статус требования, состояние реализации,
  формулировку и проверяемые критерии приёмки.
- Статус требования: «предложено», «принято», «отклонено», «устарело» или
  «заменено».
- Состояние реализации: «не выполнено», «в работе» или «выполнено».
- Изменение статуса или смысла требования фиксируется в его файле; идентификатор
  повторно не используется.
