# Promptfoo Intro (нормализованное представление)

## URL

- `https://www.promptfoo.dev/docs/intro/`

## Ключевые тезисы

- `promptfoo` представлен как open-source CLI и библиотека для evaluation и
  red teaming приложений на LLM.
- Инструмент позиционируется для проверки prompts, моделей и `RAG`, для
  автоматизированного поиска уязвимостей и для ускорения evaluation через
  caching, concurrency и live reload.
- Источник подчёркивает автоматическую оценку через определяемые metrics и
  возможность использования из CLI, библиотеки и `CI/CD`.
- Поддерживаются несколько провайдеров, включая OpenAI, Anthropic, Azure,
  Google, HuggingFace и пользовательские API.
- Базовая философия сервиса: test-driven LLM development вместо
  trial-and-error.
- Рабочий цикл состоит из пяти шагов: определить test cases, настроить
  evaluation, выполнить прогон, проанализировать результаты и пополнять набор
  примеров через feedback loop.
- Отдельно заявлены локальный режим работы и приватность: сами evals работают
  на машине пользователя и напрямую обращаются к LLM.

## Ограничения нормализации

- Нормализация основана на локальном HTML-снимке из `knowledge/primary/PRFO`.
- Полный текст страницы не переносится в Git из-за неустановленного
  лицензионного статуса.
- Маркетинговые иллюстрации, SVG-схема и детали интерфейса сведены к краткому
  пересказу.
