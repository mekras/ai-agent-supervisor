# Evaluation flows and metrics (нормализованное представление)

## URL

- `https://learn.microsoft.com/en-us/azure/machine-learning/prompt-flow/how-to-develop-an-evaluation-flow`

## Ключевые тезисы

- Источник предупреждает, что `Prompt flow` в Microsoft Foundry и Azure Machine
  Learning будет выведен из эксплуатации `20 апреля 2027 года`, а новые
  разработки рекомендуется переносить на `Microsoft Agent Framework`.
- Evaluation flow описан как особый вид prompt flow, который принимает данные и
  выходы тестируемого запуска, а затем считает оценки и метрики.
- Evaluation flow обычно запускается после основного прогона и может
  использовать `log_metric()` для записи агрегированных числовых метрик.
- Входы оценки могут включать выходы проверяемого потока, ground truth и
  дополнительный контекст; по умолчанию используется тот же набор данных, но
  его можно заменить.
- Выходы могут содержать построчные оценки по каждому примеру, а reduce-узел с
  режимом `Aggregation` позволяет посчитать итоговую оценку по всему набору.
- Итоговые metrics должны быть числовыми; строковые метрики не поддерживаются.
- Для разработки можно начать с шаблона evaluation flow или клонировать готовый
- built-in flow из галереи, а расчёт построчной оценки допускается как через
  Python-узел, так и через узел LLM.
