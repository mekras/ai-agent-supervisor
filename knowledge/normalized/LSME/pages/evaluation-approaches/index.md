# Evaluation approaches (нормализованное представление)

## URL

- `https://docs.langchain.com/langsmith/evaluation-approaches`

## Ключевые тезисы

- Страница описывает разные подходы к evaluation для популярных типов
  приложений на LLM: agents, RAG, summarization, classification и tagging.
- Для agents выделены три уровня проверки: финальный ответ агента, отдельный
  шаг агента и вся траектория.
- Проверка финального ответа рассматривает агента как чёрный ящик и часто
  использует LLM-as-judge, но хуже помогает отлаживать внутренние сбои.
- Проверка отдельного шага быстрее и лучше локализует ошибку, например выбор
  неправильного вызова инструмента или аргументов, но не покрывает полный
  агентный цикл.
- Проверка траектории оценивает последовательность шагов или вызовов
  инструментов; она
  может сравнивать точную последовательность, набор ожидаемых инструментов или
  использовать LLM-as-judge для более полного поведения.
- Для RAG отдельно рассматриваются document relevance, answer faithfulness,
  answer helpfulness, answer correctness и pairwise comparison.
- Для summarization чаще применяются reference-free критерии вроде factual
  accuracy, faithfulness и helpfulness; pairwise evaluation полезна для
  сравнения разных цепочек.
- Для classification и tagging выбор evaluation зависит от наличия reference
  labels: при наличии эталонных меток можно использовать стандартные метрики,
  без меток чаще применяют LLM-as-judge по заданным критериям.
